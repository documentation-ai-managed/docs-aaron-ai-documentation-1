## Webhooks Overview

Webhooks in Aaron AI Documentation enable real-time communication between your documentation platform and external systems. When specific events occur in your projects, Aaron AI can automatically send HTTP requests to your configured endpoints, allowing you to react to changes instantly.

## Setting Up Webhooks

Configure webhooks to receive notifications about documentation events and changes.

<Steps>
  <Step title="Create Webhook" icon="plus">
    Add a new webhook endpoint from your project settings or API.
    
    <CodeGroup tabs="JavaScript,Python">
      ````javascript
      const createWebhook = async (apiKey, webhookData) => {
        const response = await fetch('/api/webhooks', {
          method: 'POST',
          headers: {
            'Authorization': `Bearer ${apiKey}`,
            'Content-Type': 'application/json'
          },
          body: JSON.stringify(webhookData)
        });
        return response.json();
      };
      
      createWebhook('your_api_key', {
        url: 'https://your-app.com/webhook',
        events: ['document.created', 'document.updated'],
        secret: 'your_webhook_secret'
      });
      ````
      ````python
      import requests
      
      def create_webhook(api_key, webhook_data):
        response = requests.post('/api/webhooks', 
          headers={
            'Authorization': f'Bearer {api_key}',
            'Content-Type': 'application/json'
          },
          json=webhook_data)
        return response.json()
      
      create_webhook('your_api_key', {
        'url': 'https://your-app.com/webhook',
        'events': ['document.created', 'document.updated'],
        'secret': 'your_webhook_secret'
      })
      ````
    </CodeGroup>

Understand the structure and content of webhook payloads sent by Aaron AI.

<CodeGroup tabs="Example Payload,Schema">
  ````json

    "id": "wh_123456789",
    "event_type": "document.updated",
    "created_at": "2024-01-15T10:30:00Z",
    "data": {
      "document_id": "doc_987654321",

      "updated_by": "user@example.com",
      "changes": {
        "sections_added": 2,
        "sections_modified": 1,
        "word_count": 15420
      }
    },
    "signature": "sha256=abc123def456..."
  }
  ````
  ````json
  {
    "type": "object",
    "properties": {
      "id": {"type": "string"},
      "event_type": {"type": "string"},
      "created_at": {"type": "string", "format": "date-time"},
      "data": {"type": "object"},
      "signature": {"type": "string"}
    },
    "required": ["id", "event_type", "created_at", "data"]
  }
  ````

<Callout kind="alert">Always verify webhook signatures to ensure the request originates from Aaron AI and hasn't been tampered with.</Callout>

## Security Best Practices

Implement robust security measures to protect your webhook endpoints.

<ExpandableGroup>
  <Expandable title="Signature Verification" default-open="true">
    Use the webhook secret to verify that incoming requests are authentic and haven't been modified in transit.
    
    <CodeGroup tabs="Node.js,Python">
      ````javascript
      const crypto = require('crypto');
      
      function verifySignature(payload, signature, secret) {
        const expectedSignature = crypto
          .createHmac('sha256', secret)
          .update(payload, 'utf8')
          .digest('hex');
        
        return `sha256=${expectedSignature}` === signature;
      }
      
      // Usage in webhook handler
      const isValid = verifySignature(
        JSON.stringify(req.body), 
        req.headers['x-aaron-signature'], 
        process.env.WEBHOOK_SECRET
      );
      
      if (!isValid) {
        return res.status(401).send('Invalid signature');
      }
      ````
      ````python
      import hmac
      import hashlib
      
      def verify_signature(payload, signature, secret):
        expected = hmac.new(
          secret.encode('utf-8'), 
          payload.encode('utf-8'), 
          hashlib.sha256
        ).hexdigest()
        return f'sha256={expected}' == signature
      
      # Usage in webhook handler
      is_valid = verify_signature(
        json.dumps(request.get_json()), 
        request.headers.get('X-Aaron-Signature'), 
        os.getenv('WEBHOOK_SECRET')
      )
      
      if not is_valid:
        return 'Invalid signature', 401
      ````

## Error Handling and Retries

Manage webhook delivery failures and implement retry logic.

| HTTP Status | Description | Action |
|-------------|-------------|--------|
| 200-299 | Success | No retry needed |
| 400-499 | Client error | Check payload, fix endpoint |
| 500-599 | Server error | Retry with exponential backoff |

<Callout kind="tip">Implement idempotent webhook handlers to safely process duplicate events.</Callout>
